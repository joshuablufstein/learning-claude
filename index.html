Learning Claude

A mini course for non-engineers · ~28 hours · every resource free

Built for someone who already uses Claude or ChatGPT regularly and wants to close the gap between using AI well and understanding how it actually works. It assumes no coding background. If you can navigate a browser and follow written instructions, you can finish it.

It's not a class. It's a sequence of free resources — videos, short books, hands-on tutorials — arranged in the order they make most sense to consume. Each phase ends with a tangible output so you know the gap closed.

Phase 01

What an LLM actually is

~5 hours · foundational mental model

The biggest gap. You don't have a clear picture of what Claude is doing when it answers you, why it hallucinates, what training and inference mean, what a token is. Fix this first — everything else hangs on it.

  1. [1hr Talk] Intro to Large Language Models Video Andrej Karpathy · ~1 hour

    The single best plain-language explanation of the technology, by one of the people who built it. Start here.

  2. Deep Dive into LLMs like ChatGPT Video Andrej Karpathy · ~3.5 hours · optional

    Longer second pass if the Intro clicked and you want more depth. Skip if not.

  3. But what is a neural network? Video 3Blue1Brown · ~19 min

    Visual intuition for what a neural network actually computes. Pair with the next one.

  4. Gradient descent, how neural networks learn Video 3Blue1Brown · ~21 min

    Chapter 2 of the same series. How a network gets better with training, visualized.

OutputYou can explain in plain English what tokens, training, context windows, and hallucinations are — without looking anything up.

Phase 02

The plumbing

~5 hours · the layer underneath the model

What's actually happening when you open Terminal, run code, hit an API, push to a repo. Words like server, shell, package, library, port — what they really mean, with examples you can run yourself.

  1. The Missing Semester of Your CS Education Course MIT · free · do two lectures

    Built for non-engineers who need the practical scaffolding nobody teaches. Watch two specific lectures: The Shell and Version Control (Git). ~1.5 hours combined.

  2. Julia Evans' blog and zines Posts jvns.ca · free · browse

    Plain-language illustrated explanations of shell, git, networking, debugging. Browse the index, read whatever looks unfamiliar. Famous for making this stuff click.

OutputYou can define server, API, repo, package, library, terminal, shell, port, and cache in your own words.

Phase 03

Prompting and Claude specifically

~8 hours · making implicit skills explicit

You already do a lot of this by instinct. The point of this phase is making it formal, so you know what techniques you're using and when to reach for which one.

  1. Prompt Engineering Interactive Tutorial Hands-on Anthropic · ~2 hours

    The canonical resource. Free, on GitHub, with exercises you actually run. Do the whole thing.

  2. AI Fluency: Framework & Foundations Course Anthropic · free

    Higher-altitude framework for working with AI well — written by Anthropic for non-engineer professionals.

  3. Building Effective Agents Post Anthropic · ~30 min read

    The clearest writeup of when to use a single prompt vs. a workflow vs. an agent. Read it twice.

  4. Anthropic prompt engineering documentation Docs Anthropic · skim · reference

    Reference material. Skim once for the landscape, then come back when you need a specific technique.

  5. Tool use overview Docs Anthropic · skim

    How Claude calls external tools (search, calculators, APIs). The mechanism behind every agent and MCP integration.

  6. Model Context Protocol overview Docs Anthropic · skim

    The open protocol that lets Claude connect to your tools (Gmail, Notion, your file system, etc.). You'll touch this anytime you set up an integration.

OutputYou can name the prompt patterns you already use — role, structure, examples, constraints, chain-of-thought — and articulate when to reach for an agent vs. a single prompt.

Phase 04

Coding literacy and ship something

~10–15 hours · where the gap actually closes

You don't need to become a programmer. You need enough Python to read what Claude writes, modify it, and debug an error without being stuck. Then you ship one working automation so the loop is closed.

  1. Automate the Boring Stuff with Python Book Al Sweigart · free online · first 6 chapters

    Project-based Python primer written explicitly for non-engineers. Do the first six chapters — that's enough to read most scripts.

  2. n8n documentation Docs n8n · do the Quickstart

    Workflow orchestration without writing code from scratch. Do the official Quickstart, then build one real workflow that automates something you actually do.

  3. Ship one automation Project ~5–8 hours

    Pick something concrete and small: a recurring newsletter, a routine document handoff, a data pull. Build it in n8n or as a Python script with Claude's help. Read every line before you run it — no copying without understanding.

OutputYou ship one automation that runs without you, and you can read an unfamiliar Python script and roughly explain it.

Ongoing

Weekly diet

Replace AI Twitter with these two. Both free, both written by people who actually use the tools, both update on a regular cadence.

  1. One Useful Thing Newsletter Ethan Mollick · weekly-ish

    Wharton professor writing for non-engineer professionals. Pragmatic, grounded in actual experiments, no hype cycle.

  2. Simon Willison's blog Blog Simon Willison · frequent · short posts

    Developer but plain-spoken. Current, accurate, no fluff. The single best signal-to-noise source on what's actually shipping.

One principle

Every phase ends with an output you can demonstrate, not a feeling that you "learned" it. Tests, not vibes. If you can't articulate the output in your own words, you haven't finished the phase yet.
